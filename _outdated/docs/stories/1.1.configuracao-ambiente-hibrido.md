# Story 1.1: Configuração de Ambiente Híbrido

## Status
Draft

## Story
**As a** desenvolvedor da integração TeachersHub-ENEM,
**I want** configurar ambiente de desenvolvimento que suporte tanto .NET quanto Python,
**so that** eu possa trabalhar com ambos os componentes de forma integrada e eficiente.

## Acceptance Criteria
1. Docker Compose configurado with containers para TeachersHub (.NET), ENEM-API (Python), PostgreSQL e Redis
2. Variáveis de ambiente configuradas para desenvolvimento local
3. Rede Docker personalizada permitindo comunicação entre containers
4. Scripts de inicialização automatizada do ambiente completo
5. Documentação de setup atualizada com instruções passo-a-passo

## Dev Notes

### Estrutura de Repositório
[Source: docs/architecture.md#estrutura-de-repositorio]
```
enem-questions-rag/
├── teachershub-integration/     # Código .NET integração
├── python-ml-services/          # Microsserviços Python
├── shared/                      # Recursos compartilhados
│   ├── docker/                  # Dockerfiles
│   ├── database/                # Scripts SQL
│   └── monitoring/              # Configs observabilidade
├── docker-compose.yml           # Orquestração completa
└── docs/                        # Documentação técnica
```

### Arquitetura de Comunicação
[Source: docs/architecture.md#docker-compose-configuration]
- TeachersHub .NET é o orchestrador principal
- ENEM Python são microsserviços especializados ML/RAG
- PostgreSQL compartilhado com schemas separados
- Redis para cache de embeddings e performance
- Rede Docker personalizada para isolamento

### Configuração JWT
[Source: docs/architecture.md#authentication-authorization]
- JWT tokens TeachersHub devem incluir claims ENEM-específicos
- Python middleware deve validar tokens .NET
- Claims necessários: enem:access_level, enem:subjects, enem:max_questions_per_hour

### Premissas Técnicas Críticas
[Source: docs/prd.md#premissas-tecnicas]
- .NET 8, Entity Framework Core, ASP.NET Core Identity
- FastAPI, PostgreSQL, Redis para componentes Python
- Docker Compose para desenvolvimento local
- PostgreSQL único compartilhado entre componentes
- APIs RESTful com OpenAPI/Swagger
