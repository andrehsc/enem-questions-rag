## Dev Technical Guidance

### Existing System Context

**Arquitetura Híbrida Atual:**
- **TeachersHub .NET Backend** - Clean Architecture, ASP.NET Core Identity
- **Python GraphQL Services** - FastAPI com PostgreSQL, containerizado
- **Docker Compose Orchestration** - Ambiente desenvolvimento e produção
- **PostgreSQL Database** - Schema `enem_questions` com 2,532 questões

### Integration Approach

**Aditivo Strategy - Zero Breaking Changes:**

1. **New Docker Service:** `llama3-service` adicionado ao compose existente
2. **Schema Extension:** `enem_questions.question_embeddings` tabela aditiva
3. **New .NET Project:** `TeachersHub.ENEM.AI` como novo assembly
4. **Preserve APIs:** GraphQL Python mantém todas as interfaces atuais

### Technical Constraints

- **GPU Memory:** LLama3:8b requer ~8GB VRAM ou 16GB RAM sistema
- **Local Processing:** Nenhum dado pode sair do ambiente controlado
- **Performance SLA:** Resposta IA deve ser < 10s para UX aceitável
- **Database Schema:** Apenas extensões aditivas permitidas

### Configuration Examples

```yaml
# docker-compose.yml addition
llama3-service:
  image: ollama/ollama:latest
  container_name: llama3-enem-service
  volumes:
    - ./models:/root/.ollama
  ports:
    - "11434:11434"
  environment:
    - OLLAMA_HOST=0.0.0.0
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:11434/api/version"]
    interval: 30s
    timeout: 10s
    retries: 3
```

```csharp
// Program.cs integration
builder.Services.AddSemanticKernel();
builder.Services.AddScoped<IEnemAIService, EnemAIService>();
builder.Services.AddHttpClient("LLama3", client => 
{
    client.BaseAddress = new Uri("http://llama3-service:11434");
    client.Timeout = TimeSpan.FromSeconds(30);
});
```

## Dev Notes

### Critical Success Factors

1. **LLama3 Model Download** - Primeiro `docker-compose up` fará download ~4GB
2. **GPU Detection** - Verificar nvidia-docker funcionando: `docker run --gpus all nvidia/cuda:11.0-base nvidia-smi`
3. **Memory Management** - Monitor uso RAM/VRAM durante desenvolvimento
4. **Network Connectivity** - Todos containers devem se comunicar via Docker network

### Common Pitfalls

- **Model Loading Time** - LLama3 demora ~30-60s para carregar modelo
- **Vector Dimension Mismatch** - Embeddings devem ter dimensão consistente (384)
- **Database Connection Pool** - AI operations podem saturar connections
- **HTTP Timeouts** - LLama3 pode demorar, configurar timeouts adequados

### Environment Variables Required

```bash
# .env additions

LLAMA3_HOST=http://llama3-service:11434
VECTOR_EMBEDDING_DIMENSION=384
AI_RATE_LIMIT_PER_USER=10
AI_REQUEST_TIMEOUT=30
```
