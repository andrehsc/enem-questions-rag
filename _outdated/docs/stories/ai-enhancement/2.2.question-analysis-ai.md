# Story 2.2: AI-Powered Question Analysis Engine

## Status
Draft

## Story

Como professor utilizando a plataforma ENEM RAG,
Eu quero que questões sejam analisadas automaticamente por IA para identificar dificuldade, conceitos e estratégias pedagógicas,
Para que eu possa ter insights educacionais profundos sem esforço manual e otimizar meu planejamento de aulas.

## Context Source

- **Documento Base:** docs/architecture.md - "Fluxo 1: Análise Inteligente de Questões"
- **Enhancement Type:** Core AI Feature - Fase 2 (3 semanas)
- **Existing System Impact:** Extensão das capabilities GraphQL existentes

## Acceptance Criteria

**Functional Requirements:**

1. **Automatic Difficulty Analysis** - IA analisa e classifica dificuldade (1-5) para questões ENEM
2. **Concept Extraction** - Identifica automaticamente conceitos principais de cada questão
3. **Teaching Strategy Suggestions** - Gera dicas pedagógicas específicas por questão
4. **Batch Processing Capability** - Processa múltiplas questões em lote eficientemente

**Integration Requirements:**

5. **GraphQL API Extension** - Novos campos `aiAnalysis` em `QuestionType` e `QuestionSummaryType`
6. **Caching Strategy** - Análises ficam em cache para performance (PostgreSQL)
7. **Background Processing** - Análises executam assíncrono, não bloqueiam UI
8. **Existing Data Preservation** - Sistema funciona com questões sem análise IA

**Quality Requirements:**

9. **Analysis Accuracy** - 85%+ de precisão em avaliações pedagógicas (validação manual)
10. **Performance SLA** - Análise individual < 3s, batch < 30s para 10 questões
11. **Error Handling** - Fallback gracioso quando IA indisponível
12. **User Feedback Loop** - Professores podem corrigir/validar análises

## Dev Technical Guidance

### Existing System Context

- **QuestionType & QuestionSummaryType** - Tipos já estabelecidos em `api/graphql_types.py`

### Integration Approach

**Extensão Não-Destrutiva:**

1. **New AI Analysis Type:**
```graphql
type QuestionAIAnalysis {
  difficultyLevel: Int!          # 1-5 scale
  mainConcepts: [String!]!       # ["trigonometria", "equações"]
  teachingStrategies: [String!]! # ["usar exemplos visuais", ...]
  generatedAt: DateTime!
  confidence: Float!             # 0.0-1.0
}
```

2. **Extended Question Types:**
```graphql
type QuestionType {
  # ... existing fields
  aiAnalysis: QuestionAIAnalysis # null se não analisada ainda
}
```

### Technical Constraints

- **Prompt Token Limits** - LLama3 contexto máximo ~4000 tokens por request
- **Concurrent Processing** - Máximo 3 análises simultâneas para não sobrecarregar
- **Cache Strategy** - Análises válidas por 90 dias, depois re-análise
- **Error Recovery** - Sistema deve funcionar 100% sem IA ativa

### Database Schema Addition

```sql
-- Migration: Add AI Analysis Table
CREATE TABLE enem_questions.ai_analysis (
    id SERIAL PRIMARY KEY,
    question_id INTEGER REFERENCES enem_questions.questions(id),
    difficulty_level INTEGER CHECK (difficulty_level >= 1 AND difficulty_level <= 5),
    main_concepts TEXT[], -- Array of concept strings
    teaching_strategies TEXT[], -- Array of strategy strings
    confidence DECIMAL(3,2), -- 0.00 to 1.00
    model_version VARCHAR(50), -- "llama3:8b-instruct-v1"
    generated_at TIMESTAMP DEFAULT NOW(),
    validated_by_user BOOLEAN DEFAULT FALSE,
    user_feedback JSONB, -- Store corrections/validations
    
    UNIQUE(question_id, model_version) -- One analysis per model version
);

-- Indices for performance
CREATE INDEX idx_ai_analysis_question_id ON enem_questions.ai_analysis(question_id);
CREATE INDEX idx_ai_analysis_difficulty ON enem_questions.ai_analysis(difficulty_level);
CREATE INDEX idx_ai_analysis_generated_at ON enem_questions.ai_analysis(generated_at);
```

## Dev Notes

### Critical Success Factors

1. **Prompt Engineering Quality** - Templates devem ser específicos para contexto ENEM brasileiro
2. **Response Parsing Robustness** - LLama3 pode gerar JSON inconsistente, parser deve ser tolerante
3. **Cache Hit Rate** - Alto cache hit rate essencial para performance UX
4. **Background Processing** - Análises devem acontecer sem bloquear interface usuário

### Educational Context Specifics

- **ENEM Subjects:** Linguagens, Ciências Humanas, Ciências da Natureza, Matemática
- **Brazilian Context:** Prompts devem entender contexto educacional brasileiro
- **Competência-Habilidade:** Análises devem mapear para competências ENEM quando possível
- **Vocabulary Level:** Outputs devem usar terminologia pedagógica adequada para professores

### Performance Optimizations

```csharp
// Cache strategy example
private async Task<QuestionAIAnalysis> GetOrCreateAnalysisAsync(int questionId)
{
    // Try cache first (90% hit rate expected)
    var cached = await _cache.GetAnalysisAsync(questionId);
    if (cached != null && !cached.IsExpired(TimeSpan.FromDays(90)))
        return cached;
    
    // Generate new analysis
    var analysis = await GenerateAnalysisAsync(questionId);
    await _cache.StoreAnalysisAsync(questionId, analysis);
    return analysis;
}
```
