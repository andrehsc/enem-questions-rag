# Story 1.2: Real ML/RAG Models Integration

## Status
Draft

## Story
**As a** student and API user,
**I want** the RAG search and ML prediction endpoints to use real trained models,
**so that** I can get meaningful semantic search results and accurate difficulty/subject predictions for ENEM questions.

## Acceptance Criteria
1. `/rag/search` endpoint returns real semantic search results from ENEM questions
2. `/ml/predict` endpoint returns actual difficulty and subject predictions
3. RAG search uses vector embeddings for semantic similarity
4. ML models can predict question difficulty (easy/medium/hard)
5. ML models can classify question subjects with confidence scores
6. Response times are under 3 seconds for both endpoints
7. Models are containerized or accessible via API
8. Error handling for model failures with graceful degradation
9. Basic caching for frequent queries to improve performance

## Dev Notes

### Architecture Context
This story replaces the placeholder ML/RAG endpoints with real machine learning models for semantic search and prediction capabilities, enhancing the educational value of the ENEM questions API.

### Technical Implementation Details
- **Embedding Model**: sentence-transformers/multilingual models for Portuguese text
- **Vector Search**: FAISS or ChromaDB for efficient similarity search
- **ML Framework**: scikit-learn, transformers, or similar for classification
- **Caching**: Redis for embedding and prediction caching
- **Deployment**: Containerized models or external ML API services

### File Locations
Based on current project structure:
- ML utilities: `src/ml_models/` (already exists)
- RAG utilities: `src/rag_features/` (already exists)
- Model integration: `api/ml_integration.py`
- Vector database: `data/embeddings/` or external service

### Data Models
Enhance existing ML response models:
- RAGResponse: Add semantic similarity scores, related questions
- MLResponse: Add confidence scores, alternative predictions
- New types for embedding storage and retrieval

### Model Requirements
- Portuguese language support for ENEM content
- Subject classification: MATEMATICA, LINGUAGENS, CIENCIAS_HUMANAS, CIENCIAS_NATUREZA
- Difficulty levels: easy, medium, hard (based on historical data if available)
- Semantic search capability for educational content

### Testing Requirements
- Unit tests for ML pipeline components
- Integration tests for end-to-end functionality
- Performance benchmarks for response times
- Accuracy validation on sample datasets

### Technical Constraints
- Models must handle Portuguese language effectively
- Response times must be under 3 seconds
- Memory usage should be reasonable for deployment
- Fallback to basic search if ML services are unavailable
