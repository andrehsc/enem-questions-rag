# Story 3.2: Busca Semântica Integrada

## Status
Draft

## Story
**As a** professor TeachersHub,
**I want** buscar questões ENEM por conceitos pedagógicos e não apenas palavras-chave exatas,
**so that** eu encontre conteúdo mais relevante para minhas aulas com menos esforço.

## Acceptance Criteria
1. Campo de busca semântica integrado à interface TeachersHub existente, acessível na página de busca de questões
2. Resultados retornam questões semanticamente relacionadas ao conceito buscado (não apenas match de texto)
3. Cada resultado exibe explicação visual de relevância (score de similaridade em formato compreensível: Alta/Média/Baixa)
4. Busca semântica pode ser combinada com filtros tradicionais (disciplina, ano, dificuldade)
5. UI coletoa feedback do usuário (botão "Relevante" / "Não relevante") por resultado
6. Loading states durante busca e mensagens de erro amigáveis quando serviço Python indisponível

### Configuração HttpClient (.NET)
[Source: docs/architecture.md#http-client-configuration]
```csharp
// Program.cs
services.AddHttpClient<IEnemRagService>(client => {
    client.BaseAddress = new Uri(configuration["EnemServices:RagBaseUrl"]);
    client.Timeout = TimeSpan.FromSeconds(5);
}).AddResilienceHandler("rag-service", builder => {
    builder.AddRetry(new HttpRetryStrategyOptions { MaxRetryAttempts = 3 });
    builder.AddCircuitBreaker(new HttpCircuitBreakerStrategyOptions());
    builder.AddTimeout(TimeSpan.FromSeconds(5));
});
```

### Padrão de Fallback
Quando o RAG service Python estiver indisponível:
1. Circuit breaker abre após 5 falhas consecutivas
2. `EnemRagService.SearchAsync()` lança `ServiceUnavailableException`
3. Controller captura e retorna 503 com mensagem user-friendly
4. Frontend exibe banner de fallback e mantém busca simples funcional
