# Story 3.3: Sistema de Recomendações

## Status
Draft

## Story
**As a** professor TeachersHub,
**I want** receber sugestões de questões relacionadas enquanto visualizo uma questão ENEM,
**so that** eu possa descobrir conteúdo adicional relevante sem precisar fazer novas buscas.

## Acceptance Criteria
1. Seção "Questões Relacionadas" exibida na página de detalhes de cada questão com até 5 sugestões
2. Algoritmo considera contexto pedagógico (mesma disciplina e tópicos similares) e dificuldade compatível
3. Recomendações são carregadas de forma assíncrona (não bloqueiam carregamento da questão principal)
4. Professores podem marcar recomendações individualmente como "Útil" ou "Não útil"
5. Métricas de engajamento (cliques, marcações úteis/não úteis) são coletadas e armazenadas
6. Questão já visualizada na sessão atual não aparece como recomendação (deduplicação por sessão)

### Schema de Métricas
```sql
-- Schema: enem_questions
CREATE TABLE recommendation_feedback (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL,
    source_question_id INTEGER NOT NULL REFERENCES questions(id),
    recommended_question_id INTEGER NOT NULL REFERENCES questions(id),
    action VARCHAR(20) NOT NULL CHECK (action IN ('view', 'helpful', 'not_helpful')),
    session_id VARCHAR(64),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_rec_feedback_source ON recommendation_feedback(source_question_id);
CREATE INDEX idx_rec_feedback_user ON recommendation_feedback(user_id, created_at);
```

### Comportamento de Fallback
- Se RAG service indisponível: exibir seção "Questões Relacionadas" com empty state silencioso ("Sugestões indisponíveis no momento")
- NÃO bloquear carregamento da questão principal
- NÃO exibir mensagem de erro que distraia do conteúdo principal
